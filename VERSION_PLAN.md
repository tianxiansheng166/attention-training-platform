## 💡 v1.1 - v1.4 规划：专业级升级方案头脑风暴

### 第一轮头脑风暴：用户反馈分析

#### 用户痛点识别
1. **登录方式单一**：只有匿名模式，无法跨设备同步
2. **游戏数量不够**：4个游戏无法覆盖所有专注力维度
3. **家长参与不足**：没有家长视角的监控功能
4. **数据孤岛**：数据只存在本地，无法长期追踪

#### 初步改进方向
- 引入用户账户系统
- 增加更多训练游戏
- 开发家长控制台
- 加入云存储

---

### 第二轮头脑风暴：技术架构升级

#### 登录方式讨论
**问题**：手机号码或邮箱登录是否需要额外API服务？

**讨论方案**：
1. **方案A**：使用第三方身份提供商（如Auth0、Firebase Auth）
   - ✅ 优点：开发快，功能完善
   - ❌ 缺点：需要额外服务，可能产生费用，依赖第三方
   
2. **方案B**：自建邮箱/密码认证
   - ✅ 优点：完全自主，无额外服务依赖，成本低
   - ❌ 缺点：需要自己开发，需要后端

3. **方案C**：邮箱验证码登录
   - ✅ 优点：不需要记住密码，安全性较高
   - ❌ 缺点：需要邮箱服务（如阿里云邮件、SendGrid）

**最终决策**：采用 **方案B（自建邮箱/密码认证）** + **可选家长手机号绑定**
- 主要使用邮箱登录，无需额外付费API服务
- 家长端可选绑定手机号用于接收通知（使用阿里云短信）

---

#### 后端架构设计
**技术选型**：
- 后端框架：Node.js + Express.js
- 数据库：PostgreSQL（关系型数据库，适合结构化数据）
- ORM：Sequelize（简化数据库操作）
- 认证：JWT（JSON Web Token）
- 密码加密：bcryptjs
- 进程管理：PM2（生产环境）

**数据库设计**：
```
Users（用户表）
- id（主键）
- email（唯一）
- password（加密）
- parentPhone（可选）
- createdAt
- updatedAt

GameSessions（游戏会话表）
- id
- userId（外键）
- gameType
- score
- level
- duration
- accuracy
- createdAt

Assessments（评估记录表）
- id
- userId
- type
- result
- createdAt
```
